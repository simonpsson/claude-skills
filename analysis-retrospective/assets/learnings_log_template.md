# Team Learnings Log

Durable insights from retrospectives that are worth preserving across projects. Add an entry only for learnings that are specific, transferable, and not already captured in a template or reference doc.

---

## How to Add an Entry

1. Keep it short — one paragraph max per entry
2. Include the context so future readers understand when it applies
3. Note where the learning was applied (template updated, doc changed, etc.)
4. Date it so it can be reviewed for staleness later

## Quarterly Review Checklist

Run this review every quarter:

- [ ] Which entries have been applied to templates or processes?
- [ ] Which entries turned out to be wrong or too situational?
- [ ] Are there recurring patterns that suggest a bigger structural change?
- [ ] Remove or archive stale entries (older than 18 months with no reuse)
